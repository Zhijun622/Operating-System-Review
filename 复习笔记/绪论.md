## 一、概念框架（对应复习大纲“基本概念”）

### 1. 裸机 Bare Machine
- **定义**：只有硬件、尚未安装任何系统软件的计算机。
- **特点**：功能有限、只能执行机器指令、使用极不方便，需要用户直接操纵硬件（PPT L5-L67）。

### 2. 虚拟机 Virtual Machine
- **定义**：在裸机上逐层叠加 I/O 软件、文件管理、窗口系统等，形成的等价扩展计算机（教材 P3-4）。
- **作用**：屏蔽底层细节、提供统一接口，用户看到的是“扩充机器”，典型如虚拟处理机、虚拟存储器。

### 3. 操作系统 OS
- **官方定义**：控制/管理硬件与软件资源、组织工作流程并方便用户的程序集合（教材 P12）。
- **系统地位**：
  - 向上：支撑编译器、DBMS、应用等系统/应用软件；
  - 向下：统一管理 CPU、存储器、设备，屏蔽底层细节，是裸机上的第一层软件（PPT L88-L97）。
- **三大目标**（教材 P2+PPT L100-L110）：
  1. **方便性**：提供一致的用户接口（命令、GUI、系统调用）；
  2. **有效性**：提高资源利用率与系统吞吐量；
  3. **可扩充/开放性**：适应硬件、规模和互操作需求（遵循标准、支持多平台）。

## 二、OS 四大基本特征（大纲要求）

| 特征 | 教材/课程要点 |
|------|---------------|
| **并发** | 多个进程宏观并行、微观串行；引入进程概念支持多道程序在同一时间段推进（教材 P16）。 |
| **共享** | 进程对有限资源的复用：互斥共享（打印机）与同时访问（磁盘、可重入代码）。共享依赖并发，并发也以共享为前提（教材 P17）。 |
| **虚拟** | 通过时分/空分复用，把一个物理实体映射成多个逻辑实体，如虚拟处理机、虚拟内存、虚拟设备（教材 P17-19）。 |
| **异步** | 多道程序“时走时停”、执行顺序和时间不可预知，但在同步机制保证下结果可再现（教材 P19）。 |

> 口诀：**并发-共享互为条件，虚拟-异步体现效果**。四特征都以多道程序设计为技术基础。

## 三、多道程序设计技术（原理重点）

### 1. 发展动机
- 解决“CPU等待慢速 I/O”的人机矛盾，提高资源利用率和吞吐量（教材 P6-9；PPT L173-L384）。

### 2. 核心思想
- 内存同时驻留多道程序，在管理程序控制下相互穿插执行，利用进程阻塞所释放的 CPU 时间片/设备时间片（教材 P7-9，PPT L289-L363）。

### 3. 必备支撑
1. **中断技术**：为 CPU 与 I/O 建立通信机制，中断请求→响应→保存现场→处理→返回（PPT L304-L315）。
2. **通道技术**：I/O 处理机可与 CPU 并行，自动完成数据搬运（PPT L316-L326）。
3. **管理机制**：存储保护与重定位、处理机调度、资源分配与同步等（教材 P8-9）。

### 4. 多道批处理系统特征
- 多道性、宏观并行/微观串行、资源利用率高、吞吐量大，但交互性差、周转时间长（教材 P8-9）。
- 例题：三作业交错执行的时间表（PPT L328-L349）。

### 5. 对 OS 特征的贡献
- **并发**：多进程同驻内存并穿插执行；
- **共享**：进程共享 CPU、内存、I/O；
- **虚拟**：分时/分空间复用产生虚拟处理机与虚拟内存；
- **异步**：进程推进受资源竞争影响，执行不可预测。

## 四、OS 发展脉络与类型（大纲“考点内容”补充）

1. **手工操作**：无 OS，用户独占全机，CPU长期空闲（教材 P5-6）。
2. **单道批处理**：监督程序顺序执行作业，I/O 阻塞造成 CPU 空闲（教材 P6-8）。
3. **多道批处理**：引入多道程序设计，是现代 OS 的起点（教材 P7-9，PPT L289-L384）。
4. **分时系统**：多用户交互、时间片轮转，响应时间 T = n × q（教材 P9-11，PPT L398-L444）。
5. **实时系统**：对事件必须“及时/可靠”响应，分硬实时/软实时（教材 P11-13）。
6. **现代扩展**：微机 OS、嵌入式 OS、网络/分布式 OS、虚拟化等（教材 P12-15）。

## 五、重点易混概念

| 比较 | 关键点 |
|------|--------|
| **并发 vs 并行** | 并发：同一时间段；并行：同一时刻（多核）。 |
| **互斥共享 vs 同时共享** | 临界资源需要互斥；磁盘/可重入代码可“同时”被多个进程交替访问。 |
| **裸机 vs 虚拟机** | 裸机=纯硬件；虚拟机=在裸机基础上叠加 OS/软件后的抽象模型。 |
| **分时 vs 实时** | 分时强调交互与响应时间；实时强调截止时间与可靠性。 |

## 六、练习（基于本笔记，强调特征与原理理解）

### 题目1
并发性与共享性的关系正确的是（ ）
- A. 并发是共享的前提
- B. 共享是并发的前提
- C. 并发与共享互为条件
- D. 互不相关  
**答案：C**  
**解析**：没有资源共享就无法并发推进；没有并发也不会触发共享需求（教材 P17）。

### 题目2
关于虚拟性，下列说法错误的是（ ）
- A. 实现依赖时分或空分复用
- B. 虚拟处理机使进程“独占”CPU
- C. 虚拟存储器提供独立地址空间
- D. 与复用技术无关  
**答案：D**，虚拟性必须依赖分时/分空间复用（教材 P17-19）。

### 题目3
多道程序设计主要解决了（ ）之间速度不匹配的矛盾
- A. 内存与外存
- B. CPU与I/O设备
- C. 操作系统与应用程序
- D. 用户程序与系统程序  
**答案：B**（教材 P6-9，PPT L289-L349）。
